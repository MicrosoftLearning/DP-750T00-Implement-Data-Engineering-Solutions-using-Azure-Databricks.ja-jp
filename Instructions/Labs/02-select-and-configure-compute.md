# ラボ 02: Azure Databricks でコンピューティングを選択して構成する

このラボでは、あなたは **HealthBridge Analytics** でデータ エンジニアとして働いています。これは、入院や検査結果から請求とコンプライアンスのデータまで、毎日何百万もの患者レコードを処理する地域の医療機関です。 チームは、データ エンジニアリング パイプライン、アドホック分析、パイプライン検証ワークロードを実行するために Azure Databricks に依存しています。

チームがデータを処理する前に、適切なコンピューティング リソースが構成されていることを確認する必要があります。つまり、対話型開発用に適切に調整されたクラスターと、ノートブックに必要な依存関係が含まれるように適切にインストールされたライブラリが必要です。

このラボを終えるまでに、次のことを行います。

- 適切なパフォーマンス設定、Databricks Runtime、Photon Acceleration を使用して、汎用クラスターを作成して構成する。
- Databricks UI を使用してクラスター スコープ ライブラリをインストールする。
- `%pip install` を使用してノートブック スコープのライブラリをインストールし、インストールを検証する。
- インストールされているライブラリを使用して、合成患者データを生成して分析する。

このラボは完了するまで、約 **30** 分かかります。

## ノートブックをインポートする

1. Azure Databricks ワークスペースで、左側のサイド バーにある **[ワークスペース]** を選択します。

2. ラボを保存するフォルダー (個人用ホーム フォルダーなど) に移動するか、そのフォルダーを作成します。

3. フォルダーの横にある **[⋮]** (ケバブ) メニューを選択するか、フォルダーを右クリックして、**[インポート]** を選択します。

4. **[URL]** を選択し、URL (`https://raw.githubusercontent.com/MicrosoftLearning/DP-750T00-Implement-Data-Engineering-Solutions-using-Azure-Databricks/refs/heads/main/Allfiles/02-select-and-configure-compute.ipynb`) を入力し、**[インポート]** を選択します。

5. インポートしたノートブックを開きます。 これを後の手順で **[サーバーレス]** コンピューティングにアタッチします。

## 演習 1: 汎用クラスターを作成して構成する

HealthBridge のデータ エンジニアリング チームには、対話型開発用の共有クラスターが必要です。 あなたの仕事は、共同作業用の医療分析ワークロードに適した設定を使って、このクラスターを作成して構成することです。

### タスク 1.1: 新しい汎用クラスターを作成する

1. 左側のサイド バーで **[コンピュート]** を選択します。
2. **[コンピューティングの作成]** を選択します。
3. **[シンプルなフォーム]** ドロップダウンを **[オフ]** に設定します。
4. クラスター作成フォームで、次のように設定します。

    | 設定          | 値              |
    | ---------------- | ------------------ |
    | **[Cluster name](クラスター名)** | `healthbridge-dev` |
    | **ポリシー**       | 無制限       |
    | **クラスター モード** | 複数ノード         |
    | **アクセス モード**  | 共有済み             |

5. [作成] はまだ**選択しない**でください。以下の設定に進みます。

### タスク 1.2: パフォーマンスとランタイムの設定を構成する

引き続きクラスターの作成フォームで次のようにします。

1. **Databricks Runtime バージョン**で、**最新の LTS (長期サポート)** バージョンを選択します。

    > **ヒント:** LTS バージョンは、拡張サポートの対象であり、LTS 以外のバージョンよりも安定しているため、共有チーム クラスター用にお勧めします。

2. **[ワーカー タイプ]** で、**メモリ最適化**インスタンス (**E シリーズ** VM など) を選択します。 多くの場合、医療分析ワークロードには患者レコードと検査結果テーブル間の大規模な結合が含まれるため、メモリ対コア比率が高いほど有利です。

3. **[ワーカー]** で、**[自動スケーリング]** を有効にして次の設定を行います。
    - 最小ワーカー数: `1`
    - 最大ワーカー数: `3`

4. **自動終了**を有効にして、タイムアウトを **15 分**に設定します。

    > **15 分にする理由: ** これにより、分析セッションの合間にクラスターがアイドル状態になることが防がれます。これは、厳格な予算のガバナンスが求められる医療機関でのコスト管理にとって重要です。

5. 上にスクロールして、**[Photon のアクセラレーション]** が**有効になっている**ことを確認します。

6. **[コンピューティングの作成]** を選択します。

クラスターが **[実行中]** 状態に達するまで待ってから、次に進みます。 この処理には数分かかる場合があります。

## 演習 2: クラスター スコープ ライブラリをインストールする

チームは、*faker* ライブラリを使用して、パイプライン テスト用の合成患者データセットを生成します。 このライブラリを、healthbridge-dev クラスターで実行するすべてのノートブックで使用できるようにする必要があります。

[faker](https://faker.readthedocs.io/) は、名前、住所、電話番号、日付などの現実的なフェイク データを生成する Python ライブラリで、データベースのテスト、プロトタイプ作成、シード処理に使用します。 多くのロケールとデータ カテゴリをサポートしているため、実際の個人情報を使用せずに大量の妥当な合成データを簡単に生成できます。

### タスク 2.1: faker をクラスター スコープ ライブラリとしてインストールする

1. **[コンピュート]** ページで、healthbridge-dev クラスターを選択します。
2. **[ライブラリ]** タブを選択します。
3. **[新規インストール]** を選択します。
4. 次のように設定します。

    | 設定            | Value           |
    | ------------------ | --------------- |
    | **ライブラリ ソース** | PyPI            |
    | **Package**        | `faker==40.8.0` |

    > **バージョンを固定する理由: ** 医療データ エンジニアリングのコンテキストにおいては、再現性が重要です。 *faker* の厳密なバージョンを固定することで、すべてのチーム メンバーとすべてのパイプライン実行が確実に同じライブラリを使用することになり、アップストリームでのバージョンの変更による予期しない動作を防ぐことができます。

5. **[インストール]** を選択します。

6. [ライブラリ] タブにライブラリの状態が **[インストール済み]** と表示されるまで待ちます。

    > **注:** クラスター スコープのライブラリは、クラスターが再起動されるたびに自動的に再インストールされます。 これにより、手動による介入なしに環境の一貫性が保たれます。

## クラスターを停止または削除する

これでクラスターの構成とライブラリのインストールについて確認し終わったので、**healthbridge-dev** クラスターは、残りの演習にはもう必要ありません。 不要なコンピューティング コストを避けるために、続行する前にこれを停止または削除します。

1. 左側のサイド バーで **[コンピュート]** を選択します。
2. **healthbridge-dev** クラスターを選択します。
3. クラスターを停止するには、**[終了]** を選択するか、**⋮** メニューを選択し、**[削除]** を選んで完全に削除します。

    > **注:** 残りの演習は**サーバーレス** コンピューティングで実行するため、このクラスターを削除しても安全です。 削除すると、誤って再起動されて追加のコストが発生するのを防ぐことができます。

## 演習 3: ノートブックでライブラリを操作する

クラスターの構成が終わったので、ノートブックに切り替えて、残りのコード ベースの演習を完了しましょう。

### ノートブックをサーバーレス コンピューティングにアタッチする

1. 前にインポートした **02-select-and-configure-compute** ノートブックを開きます。
2. ノートブックの上部にあるコンピューティング セレクターで、**[サーバーレス]** コンピューティングを選びます。

    > **注:** ノートブックの演習 3 と 4 では、**サーバーレス** コンピューティングを使用します。 演習 1 で作成したクラスターは、コンピューティング構成を理解するうえで重要です。ノートブック自体はサーバーレスで実行され、クラスター スコープ ライブラリとは独立して動作するノートブック スコープ ライブラリのインストールを示しています。

3. ノートブック内の演習を続行します。
