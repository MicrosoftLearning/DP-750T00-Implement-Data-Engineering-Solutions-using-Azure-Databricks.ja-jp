# ラボ 10: Azure Databricks を使用してデータ パイプラインを設計および実装する

## はじめに

あなたは、**GlobStay** のデータエンジニアです。同社は、ビーチ リゾート、山小屋、市内中心部の旅館、空港のトランジット ホテルなど、5 つのホテル資産全体の予約を管理するホスピタリティ グループです。 GlobStay は夜間のバッチ パイプラインを実行して、資産管理システムから生の予約データを取り込み、それをクリーンアップして検証し、リーダーシップ チーム向けの収益分析を生成します。

このラボでは、そのパイプラインをエンドツーエンドで設計して実装します。 次の処理を行います。

- ホテル予約データ用の **medallion アーキテクチャ** (ブロンズ→シルバー→ゴールド) を構築する
- Lakeflow ジョブによって調整できる**パラメーター化された再利用可能なノートブック タスク**を記述する
- **エラー処理**を追加して、個々のタスクが失敗しても安全に停止し、その状態をダウンストリーム タスクに通知できるようにする
- タスクの依存関係、再試行ポリシー、通知を使用して**Lakeflow ジョブ**を構成する
- データ品質の結果に基づいてルーティングするための **条件付きのタスク フロー** (If/else 分岐) を調べる

## パート 1: ノートブックの演習 (medallion アーキテクチャ + エラー処理 + パラメーター)

### ノートブックをインポートする

1. Databricks ワークスペースで、左側のサイド バーにある **[ワークスペース]** をクリックします。
2. このラボを保存するフォルダーに移動するか、そのフォルダーを作成します。
3. **[⋮]** (ケバブ メニュー) をクリックするか、フォルダーを右クリックして、**[インポート]** を選択します。
4. **[URL]** を選択し、URL (`https://raw.githubusercontent.com/MicrosoftLearning/DP-750T00-Implement-Data-Engineering-Solutions-using-Azure-Databricks/refs/heads/main/Allfiles/10-design-implement-data-pipelines.ipynb`) を入力し、**[インポート]** をクリックします。

### ノートブックの演習を実行する

ノートブックには 5 つの演習が含まれています。 用意されているセットアップ セルを実行した後、各課題セルを完了します。

> **ヒント:** 手順は、課題セルのコメントに含まれています。 ✅ のマークが付いたセルは既に用意されているため、実行するだけで済みます。

## パート 2: Lakeflow ジョブを使用して調整する (UI の演習)

これらの演習では、Azure Databricks UI を使用します。 ノートブック ロジックは既に構築済みです。ここでは、これらのノートブックを運用環境レベルのマルチタスク パイプラインに合成します。

> **注:** この演習では、パート 1 でインポートしたノートブックを参照します。 実際のプロジェクトでは、レイヤーごとに個別のノートブックを作成します。 デモンストレーションの目的で、同じノートブックを異なるパラメーターで 3 回構成します。

### 演習: マルチタスク パイプライン ジョブを作成する

1. 左側のサイドバーで、**[ジョブとパイプライン]** をクリックします。
2. **[ジョブの作成]** をクリックします。
3. 上部にある既定のジョブ名を `GlobStay Booking Pipeline` に置き換えます。

#### タスク 1 - ブロンズ データを取り込む

4. タスク エディターで、タスク名 `ingest_bronze`を入力します。
5. **[種類]** ドロップダウンで、**[ノートブック]** を選択します。
6. **[ソース]** ドロップダウンで、**[ワークスペース]** を選択し、インポートしたノートブックへのパスを選択します。
7. **[コンピューティング]** フィールドで、**[サーバーレス]** を選択します。
8. **[タスクの保存]** をクリックします。

#### タスク 2 - シルバー データをクリーンする

9. タスク 1 の下部にある **[+ タスクの追加]** をクリックします (これにより、タスク 1 への依存関係が自動的に設定されます)。
10. タスクに `clean_silver` という名前を付け、**[ノートブック]**、同じノートブック パス、**[サーバーレス]** コンピューティングを入力します。
11. **[パラメーター]** で、値 `silver` を持つキー `layer` を追加します。
12. **[再試行]** で、次の設定を行います。
    - **再試行の回数**: `2`
    - **再試行の間隔:** `60` 秒
13. **[タスクの保存]** をクリックします。

#### タスク 3 - ゴールド データを集計する

14. タスク 2 の下部にある **[+ タスクの追加]** をクリックします。
15. タスクに `aggregate_gold` という名前を付け、**[ノートブック]**、同じノートブック パス、**[サーバーレス]** コンピューティングを入力します。
16. **[パラメーター]** で、値 `gold` を持つキー `layer` を追加します。
17. **[タスクの保存]** をクリックします。

#### DAG を検証する

18. DAG ビューを確認します。 次の 3 つのノードが順番に配置されています。
    ```
    ingest_bronze → clean_silver → aggregate_gold
    ```
19. タスク 2 とタスク 3 のそれぞれで、[依存先] として前のタスクが表示されていることに注意してください。

#### 失敗通知を追加する

20. ジョブ エディターの上部にある **[通知]** タブをクリックします (または、[通知] セクションまでスクロールします)。
21. **[通知の追加]** をクリックします。
22. **[失敗時]** を選択し、メール アドレスを入力します。
23. 必要に応じて、**[最後の再試行まで通知をミュートする]** を有効にして、再試行中のアラート疲労を回避します。
24. **[保存]** をクリックします。

#### ジョブを実行する

25. **[今すぐ実行]** (右上) をクリックします。
26. **[実行]** タブで、実行中の各タスクを順番に確認します。
27. 実行タイムラインでタスクをクリックすると、その出力、ログ、期間が表示されます。

### 演習: データ品質に基づくルーティングのための条件付きタスク フローを追加する

実際のパイプラインでは、データ品質の問題が検出されたかどうかに応じて異なる方法で実行をルーティングすることが必要な場合があります。 この演習では、仮定のデータ品質しきい値に基づいて分岐する **If/else 条件**タスクを追加します。

1. **[GlobStay Booking Pipeline]** ジョブ エディターに戻ります。
2. タスク 3 の横にある **[+ タスクの追加]** をクリックします (依存関係を避けるため、まず、タスク 3 の選択を解除してください)。
3. タスクに `check_quality` という名前を付け、**[ノートブック]**、同じノートブック パスを入力します。
4. このタスクを **clean_silver** に依存するように設定します。

5. **[+ タスクの追加]** をクリックします。
6. **[種類]** ドロップダウンで、**[If/else 条件]** を選択します。
7. これに `quality_gate` という名前を付けます。
8. このタスクを **check_quality** に依存するように設定します。
9. **[条件]** フィールドに次のように入力します。
   ```
   {{tasks.check_quality.values.invalid_count}} > 5
   ```
10. **[true の場合]** パスで、`alert_data_issues` という名前のタスク (ノートブック、同じパス) を追加します。
11. **[false の場合]** パスに、`proceed_gold` という名前のタスク (ノートブック、同じパス) を追加します。

> **ディスカッション:** これは、try/except を使ってノートブック自体の中でエラー処理を実装する場合と比べてどうでしょうか?  各アプローチをどんな場合に選択しますか?

## まとめ

このラボでは、次のことを行いました。

- Unity Catalog でのホテル予約データ用の **medallion アーキテクチャ** (ブロンズ → シルバー → ゴールド) の構築
- **データ クリーニング パターン** の適用: 重複除去、null フィルター処理、日付の検証、値の制約
- 資産収益と予約チャネルのパフォーマンスを網羅する**ゴールドレイヤー集計**の作成
- ジョブ レベルの通知を行うための try/except と dbutils.notebook.exit() による**エラー処理**の実装
- **dbutils.widgets** を使用したノートブックのパラメーター化と **dbutils.jobs.taskValues** を使用したタスク間での値の受け渡し
- シーケンシャル タスクの依存関係、再試行ポリシー、通知、If/else 条件タスクを使用した **Lakeflow ジョブ**の構成
